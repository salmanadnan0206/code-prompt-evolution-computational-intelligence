\chapter{Introduction}

\section{What This Project Does}

This project uses a \textbf{Genetic Algorithm (GA)} to automatically discover the best
``system prompt'' for a Large Language Model (LLM) that generates Python code.

A \textbf{system prompt} is a paragraph of plain-English instructions prepended to every
conversation with the LLM, shaping how it thinks before it sees the coding problem.
Examples: \emph{``Think like a competitive programmer''} or \emph{``Always check edge cases
before writing code.''}

The GA treats each system prompt as a \textbf{chromosome} --- a candidate solution that
can be scored (fitness evaluated), combined with another (crossover), and lightly altered
(mutation), exactly as in biological evolution.  Over 20 generations the GA evolves a
population of 15 prompts, keeping the best-performing ones and combining their strategies,
until it converges on a prompt that makes the LLM solve the most coding problems correctly.

\begin{conceptbox}{The Core Insight}
Manual prompt engineering is trial-and-error.  A human tries one prompt, tests it, tweaks
it, and tries again.  This project automates that loop: the GA \emph{searches} the entire
space of possible English instruction paragraphs by treating test-case pass rates as the
fitness signal.  The LLM's own creativity (via crossover and mutation calls) is used to
\emph{generate} new prompts, while its code-generation ability is what is being
\emph{optimised}.
\end{conceptbox}

\section{The Two-Model Architecture}

A critical design choice is that \emph{two separate Claude models} are used for two
completely different purposes:

\begin{table}[h]
\centering
\begin{tabular}{@{}llll@{}}
\toprule
\textbf{Role} & \textbf{Model} & \textbf{Temperature} & \textbf{Used for} \\
\midrule
\textbf{Target} (the subject) & \texttt{claude-haiku-4-5-20251001} & 0.0 (deterministic) & Generating Python code (fitness) \\
\textbf{Optimizer} (the tool) & \texttt{claude-sonnet-4-6}          & 0.7 (creative)      & Crossover \& mutation of prompts \\
\bottomrule
\end{tabular}
\caption{The two-model design.}
\end{table}

\begin{conceptbox}{Why Two Models?}
\textbf{Target} (\texttt{Haiku 4.5}, temp=0): We are searching for the prompt that makes
\emph{this specific model} write the best code.  Temperature 0 ensures that the same prompt
always produces the same code on the same problem, making fitness scores \emph{repeatable and
comparable} across generations.\\[6pt]
\textbf{Optimizer} (\texttt{Sonnet 4.6}, temp=0.7): Crossover and mutation require
\emph{creativity} --- merging two prompts into a coherent paragraph, inventing a new
instructional strategy.  Sonnet is more capable for reasoning tasks; temperature 0.7
introduces enough randomness to explore new wording.
\end{conceptbox}

\section{Fitness Score: What Is Being Optimised?}

The fitness of one prompt = the average partial-credit score across 100 APPS problems.
The partial credit for one problem = (test cases passed) / (total test cases).

\[
\text{fitness}(p) = \frac{1}{100} \sum_{i=1}^{100}
  \frac{\text{tests passed on problem } i \text{ using prompt } p}{\text{total tests on problem } i}
\]

A fitness of 1.0 means the prompt caused the LLM to write code that passed \emph{every}
test case on \emph{every} problem.  A fitness of 0.73 means 73\% of test cases were passed
on average.

import os

# Paths

# Dataset

HOLDOUT_SIZE   = 200   # held-out for final within-distribution test
RANDOM_SEED    = 42

# Genetic Algorithm
POPULATION_SIZE   = 15
GENERATIONS       = 20
ELITISM_COUNT     = 2   # top-N individuals carried unchanged to next generation
TOURNAMENT_SIZE   = 3   # contestants per tournament selection draw

\subsection{Genetic Algorithm Constants (lines 22--28)}

\begin{conceptbox}{Elitism prevents regression}
Without elitism, a great prompt discovered in generation 5 might be replaced by worse
offspring in generation 6.  \fn{ELITISM\_COUNT = 2} guarantees the top 2 prompts always
survive into the next generation \emph{unchanged}, so the population's best fitness can
only stay the same or improve.
\end{conceptbox}

\begin{conceptbox}{Tournament size: exploration vs.\ exploitation}
\fn{TOURNAMENT\_SIZE = 3} means we draw 3 random contestants and select the best.
A smaller tournament (e.g.\ 2) gives weaker individuals more chances, increasing
exploration.  A larger tournament (e.g.\ 10) almost always picks the global best,
reducing exploration.  Size 3 is a standard balanced choice.
\end{conceptbox}

\section{Purpose}

instruction prompts --- the ``generation zero'' chromosomes.  Each emphasises a

of novelty.  By seeding 15 different strategies --- edge-case thinking, algorithm
identification, pseudocode-first, output-format checking, etc.\ --- crossover can
combine genuinely different approaches.  A hybrid of ``analyse edge cases first'' and
``identify the algorithm pattern'' can be better than either parent alone.

"""
dataset.py -- Load and split APPS Codewars introductory fn_name problems.

Why Codewars only?
    * 2,346 introductory problems from codewars.com, all standalone functions.
    * Consistent execution format:
        inputs[i]  = Python list of arguments already parsed (e.g. [1] or [2, "x"])
        outputs[i] = [return_value]  -- the result wrapped in a single-element list
    * LeetCode-style problems use class Solution, which complicates extraction.
"""

import json
import random
import sys
from pathlib import Path


def load_problems(apps_train_dir: str | Path) -> list[dict]:
    """
    Return every Codewars introductory fn_name problem from the APPS train set.

    Each problem dict has:
        id        - folder name (e.g. "3715")
        question  - full problem text
        fn_name   - function the LLM must write
        inputs    - list of test-case argument lists
        outputs   - list of [expected_return_value]
    """
    sys.set_int_max_str_digits(1_000_000)   # needed for a handful of problems
    problems = []
    train_dir = Path(apps_train_dir)

    for prob_dir in train_dir.iterdir():
        meta_f = prob_dir / "metadata.json"
        io_f   = prob_dir / "input_output.json"
        q_f    = prob_dir / "question.txt"

        if not (meta_f.exists() and io_f.exists() and q_f.exists()):
            continue

        try:
            meta = json.loads(meta_f.read_text())
            io   = json.loads(io_f.read_text())
        except Exception:
            continue   # skip malformed JSON

        # Keep only: introductory difficulty, fn_name style, Codewars source
        if (
            meta.get("difficulty") != "introductory"
            or "fn_name" not in io
            or "codewars.com" not in meta.get("url", "")
            or not io.get("inputs")
            or not io.get("outputs")
        ):
            continue

        problems.append({
            "id":       prob_dir.name,
            "question": q_f.read_text().strip(),
            "fn_name":  io["fn_name"],
            "inputs":   io["inputs"],
            "outputs":  io["outputs"],
        })

    return problems


def split(
    problems: list[dict],
    evolution_size: int,
    holdout_size: int,
    seed: int = 42,
) -> tuple[list[dict], list[dict], list[dict]]:
    """
    Shuffle and split into:
        evolution -- used for fitness evaluation every generation
        holdout   -- unseen until final evaluation
        rest      -- remaining problems (not used during training)
    """
    rng = random.Random(seed)
    shuffled = problems.copy()
    rng.shuffle(shuffled)

    evolution = shuffled[:evolution_size]
    holdout   = shuffled[evolution_size : evolution_size + holdout_size]
    rest      = shuffled[evolution_size + holdout_size :]

    return evolution, holdout, rest
\end{lstlisting}

Execution model (Codewars APPS fn_name problems):
    inputs[i]  = Python list already parsed, e.g. [1] or ["hello", 2]
    outputs[i] = [expected_return_value], e.g. [5] or [True]

    result = fn_name(*inputs[i])
    passed = ([result] == outputs[i])

Fitness = (test cases passed) / (total test cases)   -- partial credit.
"""

import re
import subprocess
import sys
from typing import Optional


# ---------------------------------------------------------------------------
# Code extraction
# ---------------------------------------------------------------------------

def extract_code(response: str) -> str:
    """
    Pull the Python code block out of an LLM response.
    Tries ```python ... ``` first, then any ``` ... ```, then the raw text.
    """
    # Markdown fenced code block with python tag
    m = re.search(r"```python\s*\n([\s\S]*?)```", response)
    if m:
        return m.group(1).strip()

    # Any fenced code block
    m = re.search(r"```\s*\n?([\s\S]*?)```", response)
    if m:
        return m.group(1).strip()

    # No fences -- assume the whole response is code
    return response.strip()


def has_function(code: str, fn_name: str) -> bool:
    """Return True if `def fn_name(` appears in the code."""
    return bool(re.search(rf"\bdef\s+{re.escape(fn_name)}\s*\(", code))


# ---------------------------------------------------------------------------
# Subprocess test runner
# ---------------------------------------------------------------------------

_HARNESS_TEMPLATE = """\
import sys
sys.set_int_max_str_digits(1_000_000)

{code}

def __run__(inp, expected):
    try:
        result = {fn_name}(*inp)
        sys.stdout.write("__R__:" + ("1" if [result] == expected else "0") + "\\n")
        sys.stdout.flush()
    except Exception:
        sys.stdout.write("__R__:0\\n")
        sys.stdout.flush()

{calls}
"""


def _build_script(code: str, fn_name: str, inputs: list, outputs: list) -> str:
    """Build the standalone test script that will be executed in a subprocess."""
    call_lines = "\n".join(
        f"__run__({repr(inp)}, {repr(out)})"
        for inp, out in zip(inputs, outputs)
    )
    return _HARNESS_TEMPLATE.format(
        code=code,
        fn_name=fn_name,
        calls=call_lines,
    )


def run_tests(
    code: str,
    fn_name: str,
    inputs: list,
    outputs: list,
    timeout: int = 5,
) -> float:
    """
    Execute generated code against all test cases in an isolated subprocess.

    Returns partial-credit score: (passed tests) / (total tests).
    Returns 0.0 on any execution error, timeout, or missing function.
    """
    if not code or not has_function(code, fn_name):
        return 0.0

    script = _build_script(code, fn_name, inputs, outputs)

    try:
        proc = subprocess.run(
            [sys.executable, "-c", script],
            capture_output=True,
            text=True,
            timeout=timeout,
        )
        # Parse only our sentinel lines to ignore any stray print() in generated code
        results = [
            int(line[len("__R__:"):])
            for line in proc.stdout.splitlines()
            if line.startswith("__R__:")
        ]
        if not results:
            return 0.0
        return sum(results) / len(inputs)   # partial credit over ALL test cases

    except subprocess.TimeoutExpired:
        return 0.0
    except Exception:
        return 0.0
\end{lstlisting}

LLMs frequently wrap code in Markdown fences.  The function tries three strategies in
order, returning as soon as one succeeds:

\textbf{Strategy 1 --- tagged python fence} (lines 29--31):
\begin{lstlisting}[style=example]
# Regex: r"```python\s*\n([\s\S]*?)```"
# Matches:  ```python
#           <any content, including newlines ([\s\S]*?)>
#           ```
# Group 1 = the code inside the fences (captured by the first parentheses)
\end{lstlisting}

\textbf{Strategy 2 --- untagged fence} (lines 33--35):
\begin{lstlisting}[style=example]
# Regex: r"```\s*\n?([\s\S]*?)```"
# Matches: ``` (with optional whitespace / newline before content)
# Handles responses like:
#   ```
#   def foo(): ...
#   ```
\end{lstlisting}

\textbf{Strategy 3 --- raw text} (line 38):
If no fences are found, assume the entire response is already plain code (some prompts
cause the LLM to output code directly with no markdown).

\begin{lstlisting}[style=example]

\textbf{Defined:} \pyfile{evaluator.py}, line 43\\
\textbf{Called by:} \fn{run\_tests()} [evaluator.py:97]\\
\textbf{Calls:} \fn{re.search()}, \fn{re.escape()}\\
\textbf{Parameters:}\\
\hspace{1em}\param{code} --- extracted Python code string\\
\hspace{1em}\param{fn\_name} --- the function name the LLM was asked to write (e.g., \texttt{"nth\_char"})\\
\textbf{Returns:} \texttt{bool} --- \texttt{True} if \texttt{def fn\_name(} appears in \param{code}
\end{specbox}

\begin{lstlisting}[style=example]
# Regex: rf"\bdef\s+{re.escape(fn_name)}\s*\("
#
# \b          = word boundary (prevents matching "def foo_bar(" when fn_name="bar")
# def         = literal keyword
# \s+         = one or more spaces/tabs
# fn_name     = exact function name (re.escape handles special chars like ".")
# \s*         = zero or more spaces before "("
# \(          = opening parenthesis

has_function("def nth_char(words):\n    return ...", "nth_char")  # True
has_function("def nth_char(words):\n    return ...", "sum_list")  # False
has_function("# nth_char defined elsewhere", "nth_char")          # False (no "def ")
has_function("class Solution:\n    def nth_char(self): ...", "nth_char") # True
\end{lstlisting}

If \fn{has\_function} returns \texttt{False}, \fn{run\_tests} immediately returns 0.0
without attempting execution.  This guards against the LLM writing the wrong function
name, which would cause an unresolvable \texttt{NameError} in the subprocess.

\subsection{\fn{\_HARNESS\_TEMPLATE}}

\begin{specbox}{\_HARNESS\_TEMPLATE: str (module-level constant)}
\textbf{Defined:} \pyfile{evaluator.py}, line 52\\
\textbf{Used by:} \fn{\_build\_script()} to produce the complete subprocess script\\
\textbf{Template variables:} \texttt{\{code\}}, \texttt{\{fn\_name\}}, \texttt{\{calls\}}
\end{specbox}

The template defines a small Python program that will be run in a subprocess:

\begin{lstlisting}[style=example]
# _HARNESS_TEMPLATE (with placeholders shown):
import sys
sys.set_int_max_str_digits(1_000_000)  # same large-int fix as dataset.py

{code}                    # <-- the LLM-generated function pasted verbatim

def __run__(inp, expected):
    try:
        result = {fn_name}(*inp)    # <-- calls the generated function
        # [result] wraps return value in list to match APPS output format
        sys.stdout.write("__R__:" + ("1" if [result] == expected else "0") + "\n")
        sys.stdout.flush()
    except Exception:               # <-- catches ALL exceptions (NameError, TypeError...)
        sys.stdout.write("__R__:0\n")
        sys.stdout.flush()

{calls}    # <-- one __run__(...) call per test case
\end{lstlisting}

\begin{conceptbox}{Why \texttt{[result] == expected}?}
Codewars APPS problems store expected outputs as \texttt{[return\_value]} (a list
containing the single return value).  For example, if \fn{nth\_char} should return
\texttt{"oes"}, then \texttt{outputs[i] = ["oes"]}.  Wrapping the actual result in
\texttt{[result]} makes the comparison format match: \texttt{["oes"] == ["oes"]} is
\texttt{True}; \texttt{["oes"] == ["abc"]} is \texttt{False}.\\[4pt]
Without the wrapping: \texttt{"oes" == ["oes"]} would always be \texttt{False} even for
correct code.
\end{conceptbox}

\begin{conceptbox}{Why a subprocess?}
The LLM-generated code is untrusted.  Importing it directly into the GA process could:
\begin{itemize}[nosep]
  \item Crash the entire GA (via a top-level \texttt{sys.exit()} or segfault).
  \item Corrupt global state (e.g., monkey-patching Python builtins).
  \item Hang indefinitely (infinite loop with no timeout mechanism).
\end{itemize}
Running in a subprocess isolates all of this: if the subprocess crashes, hangs, or
produces garbage, the GA process is completely unaffected.
\end{conceptbox}

\textbf{Step-by-step walkthrough:}

\begin{enumerate}
  \item \textbf{Lines 97--98}: Early return 0.0 if \param{code} is empty (the LLM returned
    nothing) or if \fn{has\_function} returns False (the required function is not defined).

  \item \textbf{Lines 100}: Call \fn{\_build\_script} to assemble the executable test script.

  \item \textbf{Lines 102--108}: Launch the script in an isolated subprocess:
    \begin{lstlisting}[style=example]
proc = subprocess.run(
    [sys.executable, "-c", script],  # python3 -c "..."  (full script as argument)
    capture_output=True,              # capture stdout and stderr
    text=True,                        # decode bytes to str automatically
    timeout=timeout,                  # kill process after 5 seconds
)
    \end{lstlisting}

  \item \textbf{Lines 109--114}: Parse \texttt{proc.stdout} for sentinel lines:
    \begin{lstlisting}[style=example]
results = [
    int(line[len("__R__:"):])          # "1" or "0" after the prefix
    for line in proc.stdout.splitlines()
    if line.startswith("__R__:")       # ignore all other lines (stray prints)
]
# If stdout = "__R__:1\n__R__:1\n__R__:0\n__R__:1\n"
# results  = [1, 1, 0, 1]
    \end{lstlisting}

  \item \textbf{Lines 115--116}: If \fn{results} is empty (script crashed on load ---
    e.g.\ syntax error), return 0.0.

  \item \textbf{Line 117}: Return partial-credit score:
    \begin{lstlisting}[style=example]
return sum(results) / len(inputs)
#          ^-- passed tests          ^-- TOTAL test cases (not just those that ran)

# Example: sum([1,1,0,1]) / 4 = 3/4 = 0.75
# If script crashes after 2 tests: results=[1,1], len(inputs)=4 => 0.5
# (not 1.0, because the 2 un-run tests count as failures in the denominator)
    \end{lstlisting}

  \item \textbf{Lines 118--121}: Exception handling:
    \begin{itemize}[nosep]
      \item \fn{subprocess.TimeoutExpired} --- the 5-second limit was hit; return 0.0.
      \item Any other exception --- subprocess module error; return 0.0.
    \end{itemize}
\end{enumerate}

\begin{warnbox}{Division by \texttt{len(inputs)}, not \texttt{len(results)}}
The denominator is \texttt{len(inputs)} (total test cases) not \texttt{len(results)}
(test cases that produced output).  If generated code crashes midway through the tests,
the un-run tests count as failures.  This correctly penalises fragile code that only
works on some inputs before crashing.
\end{warnbox}

\begin{conceptbox}{The Sentinel Design}
Generated code may contain stray \texttt{print()} statements (the LLM might print debug
output like ``Input is 5'').  These would mix into \texttt{proc.stdout} and corrupt the
results if we tried to parse every line.  By only looking at lines that start with the
unique prefix \texttt{\_\_R\_\_:}, we ignore all other output completely.  The prefix was
chosen to be unlikely to appear in any realistic program output.
\end{conceptbox}

"""
cost_tracker.py -- Track every API call's token usage and compute running cost.
"""

import json
import time
from dataclasses import dataclass, field
from datetime import datetime
from pathlib import Path

# Official Anthropic pricing (USD per 1 million tokens) -- April 2026
PRICING = {
    "claude-haiku-4-5-20251001": (1.00,  5.00),
    "claude-haiku-3-5-20241022": (0.80,  4.00),
    "claude-haiku-3-20240307":   (0.25,  1.25),
    "claude-sonnet-4-6":         (3.00, 15.00),
    "claude-sonnet-4-5-20250514":(3.00, 15.00),
    "claude-sonnet-4-20250514":  (3.00, 15.00),
    "claude-opus-4-6":           (5.00, 25.00),
    # Batch API (50% discount)
    "claude-haiku-4-5-20251001:batch": (0.50, 2.50),
    "claude-sonnet-4-6:batch":         (1.50, 7.50),
}

COSTS_DIR = Path(__file__).parent / "Costs"


@dataclass
class CallRecord:
    phase: str          # "fitness", "crossover", "mutate_inject", etc.
    model: str
    input_tokens: int
    output_tokens: int

    @property
    def input_cost(self) -> float:
        rate_in, _ = PRICING.get(self.model, (0, 0))
        return self.input_tokens * rate_in / 1_000_000

    @property
    def output_cost(self) -> float:
        _, rate_out = PRICING.get(self.model, (0, 0))
        return self.output_tokens * rate_out / 1_000_000

    @property
    def total_cost(self) -> float:
        return self.input_cost + self.output_cost


class CostTracker:
    """Accumulates CallRecords and saves a full breakdown JSON on flush."""

    def __init__(self):
        self.calls: list[CallRecord] = []
        self.start_time: float = time.time()

    def record(self, phase: str, model: str,
               input_tokens: int, output_tokens: int):
        """Log one API call."""
        self.calls.append(CallRecord(
            phase=phase, model=model,
            input_tokens=input_tokens,
            output_tokens=output_tokens,
        ))

    def _by_phase(self) -> dict:
        phases: dict[str, dict] = {}
        for c in self.calls:
            if c.phase not in phases:
                phases[c.phase] = {
                    "calls": 0, "input_tokens": 0, "output_tokens": 0,
                    "input_cost_usd": 0.0, "output_cost_usd": 0.0,
                    "total_cost_usd": 0.0,
                }
            p = phases[c.phase]
            p["calls"]           += 1
            p["input_tokens"]    += c.input_tokens
            p["output_tokens"]   += c.output_tokens
            p["input_cost_usd"]  += c.input_cost
            p["output_cost_usd"] += c.output_cost
            p["total_cost_usd"]  += c.total_cost
        for p in phases.values():
            for k in ("input_cost_usd", "output_cost_usd", "total_cost_usd"):
                p[k] = round(p[k], 6)
        return phases

    def _by_model(self) -> dict:
        models: dict[str, dict] = {}
        for c in self.calls:
            if c.model not in models:
                models[c.model] = {
                    "calls": 0, "input_tokens": 0, "output_tokens": 0,
                    "input_cost_usd": 0.0, "output_cost_usd": 0.0,
                    "total_cost_usd": 0.0,
                }
            m = models[c.model]
            m["calls"]           += 1
            m["input_tokens"]    += c.input_tokens
            m["output_tokens"]   += c.output_tokens
            m["input_cost_usd"]  += c.input_cost
            m["output_cost_usd"] += c.output_cost
            m["total_cost_usd"]  += c.total_cost
        for m in models.values():
            for k in ("input_cost_usd", "output_cost_usd", "total_cost_usd"):
                m[k] = round(m[k], 6)
        return models

    def total(self) -> float:
        return sum(c.total_cost for c in self.calls)

    def flush(self, extra: dict | None = None) -> Path:
        """Write a timestamped JSON to the Costs/ folder."""
        COSTS_DIR.mkdir(parents=True, exist_ok=True)
        ts = datetime.now().strftime("%Y%m%d_%H%M%S")
        path = COSTS_DIR / f"cost_{ts}.json"

        total_in  = sum(c.input_tokens for c in self.calls)
        total_out = sum(c.output_tokens for c in self.calls)
        elapsed   = time.time() - self.start_time

        report = {
            "timestamp":        ts,
            "elapsed_seconds":  round(elapsed, 1),
            "total_api_calls":  len(self.calls),
            "total_input_tokens":  total_in,
            "total_output_tokens": total_out,
            "total_input_cost_usd":  round(sum(c.input_cost  for c in self.calls), 6),
            "total_output_cost_usd": round(sum(c.output_cost for c in self.calls), 6),
            "grand_total_usd":       round(self.total(), 6),
            "breakdown_by_phase": self._by_phase(),
            "breakdown_by_model": self._by_model(),
            "pricing_used": {
                model: {"input_per_MTok": r[0], "output_per_MTok": r[1]}
                for model, r in PRICING.items()
                if not model.endswith(":batch")
            },
        }
        if extra:
            report["run_info"] = extra

        path.write_text(json.dumps(report, indent=2))
        return path


# Module-level singleton imported by llm.py
tracker = CostTracker()

"""
llm.py -- All Claude API calls: code generation, crossover, and mutation.

Two models are used:
    TARGET_MODEL    -- generates code (cheap, deterministic, temperature 0)
    OPTIMIZER_MODEL -- performs crossover and mutation (creative, temperature 0.7)

Every call is logged to the CostTracker singleton (see cost_tracker.py).
"""

import re
import time
import anthropic
import config
from cost_tracker import tracker


_client: anthropic.Anthropic | None = None


def _get_client() -> anthropic.Anthropic:
    """Lazy-initialise the Anthropic client."""
    global _client
    if _client is None:
        _client = anthropic.Anthropic(api_key=config.ANTHROPIC_API_KEY)
    return _client


def _call(
    phase: str,
    model: str,
    temperature: float,
    max_tokens: int,
    system: str,
    user: str,
    retries: int = 3,
) -> str:
    """
    Call the Claude API with automatic retry on transient errors.
    Logs token usage to the global CostTracker.
    Returns the assistant's text response.
    """
    client = _get_client()
    for attempt in range(retries):
        try:
            resp = client.messages.create(
                model=model,
                max_tokens=max_tokens,
                temperature=temperature,
                system=system,
                messages=[{"role": "user", "content": user}],
            )
            # Log tokens to cost tracker
            tracker.record(
                phase=phase,
                model=model,
                input_tokens=resp.usage.input_tokens,
                output_tokens=resp.usage.output_tokens,
            )
            return resp.content[0].text
        except anthropic.RateLimitError:
            wait = 2 ** attempt * 5
            print(f"  [rate-limit] waiting {wait}s ...")
            time.sleep(wait)
        except anthropic.APIError as e:
            if attempt == retries - 1:
                print(f"  [api-error] {e}")
                return ""
            time.sleep(2)
    return ""


# ---------------------------------------------------------------------------
# Code generation  (fitness evaluation)
# ---------------------------------------------------------------------------

def generate_code(system_prompt: str, question: str, fn_name: str) -> str:
    """
    Ask the target LLM to solve a coding problem.
    Returns the raw LLM response (code extraction happens in evaluator.py).
    """
    user_msg = (
        f"{question}\n\n"
        f"Write a Python function named `{fn_name}`."
    )
    return _call(
        phase="fitness",
        model=config.TARGET_MODEL,
        temperature=config.TARGET_TEMPERATURE,
        max_tokens=config.MAX_CODE_TOKENS,
        system=system_prompt,
        user=user_msg,
    )


# ---------------------------------------------------------------------------
# Crossover  (LLM-mediated merge of two parent prompts)
# ---------------------------------------------------------------------------

_CROSSOVER_SYSTEM = (
    "You merge coding-instruction prompts. Output ONLY the merged prompt -- "
    "no commentary, no labels, no markdown."
)

_CROSSOVER_USER = """\
Combine these two coding-instruction prompts into one coherent prompt.
Preserve the strongest strategies from each parent (e.g. edge-case thinking
from one, algorithm identification from the other).
Do not simply concatenate -- produce a unified instruction paragraph.

--- Parent A ---
{parent_a}

--- Parent B ---
{parent_b}
"""


def crossover(prompt_a: str, prompt_b: str) -> str:
    """Merge two parent prompts into one offspring via the optimizer LLM."""
    return _call(
        phase="crossover",
        model=config.OPTIMIZER_MODEL,
        temperature=config.OPTIMIZER_TEMPERATURE,
        max_tokens=config.MAX_OPERATOR_TOKENS,
        system=_CROSSOVER_SYSTEM,
        user=_CROSSOVER_USER.format(parent_a=prompt_a, parent_b=prompt_b),
    ).strip()


# ---------------------------------------------------------------------------
# Mutation  (three independent operators)
# ---------------------------------------------------------------------------

_MUT_SYSTEM = (
    "You edit coding-instruction prompts. Output ONLY the modified prompt -- "
    "no commentary, no labels, no markdown."
)


def mutate_inject(prompt: str) -> str:
    """Add one new instructional strategy to the prompt."""
    user = (
        "Add one new, specific coding strategy to this instruction prompt. "
        "Examples of strategies: 'validate your output against the examples', "
        "'consider time complexity', 'check for off-by-one errors'. "
        "Output only the modified prompt.\n\n"
        f"Prompt:\n{prompt}"
    )
    return _call(
        phase="mutate_inject",
        model=config.OPTIMIZER_MODEL,
        temperature=config.OPTIMIZER_TEMPERATURE,
        max_tokens=config.MAX_OPERATOR_TOKENS,
        system=_MUT_SYSTEM,
        user=user,
    ).strip()


def mutate_delete(prompt: str) -> str:
    """Remove one sentence from the prompt to make it leaner."""
    sentences = re.split(r"(?<=[.!?])\s+", prompt)
    if len(sentences) <= 1:
        return prompt  # nothing to delete -- keep as-is
    user = (
        "Remove one sentence from this instruction prompt -- the one that "
        "contributes the least to code-generation quality. Keep everything "
        "else verbatim. Output only the modified prompt.\n\n"
        f"Prompt:\n{prompt}"
    )
    return _call(
        phase="mutate_delete",
        model=config.OPTIMIZER_MODEL,
        temperature=config.OPTIMIZER_TEMPERATURE,
        max_tokens=config.MAX_OPERATOR_TOKENS,
        system=_MUT_SYSTEM,
        user=user,
    ).strip()


def mutate_rephrase(prompt: str) -> str:
    """Rephrase the prompt without changing its meaning."""
    user = (
        "Rephrase this instruction prompt using different words, but keep "
        "the same meaning and all the same strategies. "
        "Output only the rephrased prompt.\n\n"
        f"Prompt:\n{prompt}"
    )
    return _call(
        phase="mutate_rephrase",
        model=config.OPTIMIZER_MODEL,
        temperature=config.OPTIMIZER_TEMPERATURE,
        max_tokens=config.MAX_OPERATOR_TOKENS,
        system=_MUT_SYSTEM,
        user=user,
    ).strip()
\end{lstlisting}

\begin{conceptbox}{What happens when \_call returns \texttt{""}?}
If \fn{\_call} returns an empty string, the caller (e.g.\ \fn{generate\_code}) passes
it to \fn{evaluator.extract\_code("")} which returns \texttt{""}, then
\fn{evaluator.run\_tests("")} returns 0.0 immediately (the empty-code guard).
The individual gets fitness 0.0 for that problem.  The run continues --- a failed
API call does \emph{not} crash the GA.
\end{conceptbox}

\begin{conceptbox}{Crossover as semantic blending}

parent prompts and creates a semantically coherent child.  The meta-prompt explicitly
forbids simple concatenation --- the LLM must integrate the strategies, resolving
conflicts and removing redundancies.
\end{conceptbox}

"""
ga.py -- Genetic Algorithm for evolving system-level instruction prompts.

Pipeline summary (one generation):
    1. SELECTION  -- tournament (size 3), pick 2 parents
    2. CROSSOVER  -- LLM merges parents (rate 0.8)
    3. MUTATION   -- inject / delete / rephrase (independent)
    4. FITNESS    -- run code on 100 APPS problems
    5. REPLACEMENT -- generational with top-2 elitism
"""

from __future__ import annotations

import random
import uuid
from dataclasses import dataclass, field

import config
import evaluator
import llm


# ---------------------------------------------------------------------------
# Individual
# ---------------------------------------------------------------------------

@dataclass
class Individual:
    """One chromosome = one system-level instruction prompt."""
    prompt: str
    fitness: float = -1.0
    uid: str = field(default_factory=lambda: uuid.uuid4().hex[:8])
    parent_a: str = ""      # uid of parent A (empty for seeds)
    parent_b: str = ""      # uid of parent B
    operators: str = ""     # operators applied, e.g. "crossover+inject"


# ---------------------------------------------------------------------------
# Fitness
# ---------------------------------------------------------------------------

def compute_fitness(individual: Individual, problems: list[dict]) -> float:
    """
    Evaluate a prompt on the evolution pool.
    Fitness = mean score across all problems (partial credit per problem).
    """
    total = 0.0
    for prob in problems:
        response = llm.generate_code(
            system_prompt=individual.prompt,
            question=prob["question"],
            fn_name=prob["fn_name"],
        )
        code = evaluator.extract_code(response)
        score = evaluator.run_tests(
            code=code,
            fn_name=prob["fn_name"],
            inputs=prob["inputs"],
            outputs=prob["outputs"],
            timeout=config.EXEC_TIMEOUT,
        )
        total += score
    return total / len(problems)


# ---------------------------------------------------------------------------
# Selection
# ---------------------------------------------------------------------------

def tournament_select(population: list[Individual], rng: random.Random) -> Individual:
    """Pick TOURNAMENT_SIZE random individuals, return the fittest."""
    contestants = rng.sample(population, config.TOURNAMENT_SIZE)
    return max(contestants, key=lambda ind: ind.fitness)


# ---------------------------------------------------------------------------
# Crossover + mutation -- one offspring
# ---------------------------------------------------------------------------

def breed(
    parent_a: Individual,
    parent_b: Individual,
    rng: random.Random,
) -> Individual:
    """
    Produce one offspring from two parents.
    Returns an unevaluated Individual (fitness = -1).
    """
    ops = []

    # --- crossover ---
    if rng.random() < config.CROSSOVER_RATE:
        prompt = llm.crossover(parent_a.prompt, parent_b.prompt)
        ops.append("crossover")
    else:
        prompt = rng.choice([parent_a.prompt, parent_b.prompt])
        ops.append("copy")

    # --- mutations (independent -- all three can fire) ---
    if rng.random() < config.MUT_INJECT_PROB:
        prompt = llm.mutate_inject(prompt)
        ops.append("inject")
    if rng.random() < config.MUT_DELETE_PROB:
        prompt = llm.mutate_delete(prompt)
        ops.append("delete")
    if rng.random() < config.MUT_REPHRASE_PROB:
        prompt = llm.mutate_rephrase(prompt)
        ops.append("rephrase")

    return Individual(
        prompt=prompt,
        parent_a=parent_a.uid,
        parent_b=parent_b.uid,
        operators="+".join(ops),
    )


# ---------------------------------------------------------------------------
# Main GA loop
# ---------------------------------------------------------------------------

def run(
    seed_prompts: list[str],
    problems: list[dict],
    on_generation=None,
) -> tuple[Individual, list[dict]]:
    """
    Run the full genetic algorithm.
    Returns: (best_individual, history)
    """
    rng = random.Random(config.RANDOM_SEED)
    history: list[dict] = []

    # --- initialise population ---
    population = [Individual(prompt=p) for p in seed_prompts]
    print(f"[gen 0] evaluating {len(population)} seed individuals ...")
    for ind in population:
        ind.fitness = compute_fitness(ind, problems)
        print(f"  {ind.uid}  fitness={ind.fitness:.4f}")

    for gen in range(1, config.GENERATIONS + 1):
        # --- record stats for PREVIOUS generation ---
        population.sort(key=lambda x: x.fitness, reverse=True)
        fits = [x.fitness for x in population]
        stats = {
            "generation": gen - 1,
            "best": max(fits),
            "mean": sum(fits) / len(fits),
            "worst": min(fits),
            "best_uid": population[0].uid,
            "best_prompt": population[0].prompt,
        }
        history.append(stats)
        print(
            f"[gen {gen-1}] best={stats['best']:.4f}  "
            f"mean={stats['mean']:.4f}  worst={stats['worst']:.4f}"
        )

        if on_generation:
            on_generation(gen - 1, population, stats)

        # --- elitism: carry over top ELITISM_COUNT ---
        new_pop = population[: config.ELITISM_COUNT]

        # --- breed offspring to fill remaining slots ---
        while len(new_pop) < config.POPULATION_SIZE:
            p1 = tournament_select(population, rng)
            p2 = tournament_select(population, rng)
            child = breed(p1, p2, rng)
            new_pop.append(child)

        # --- evaluate only the new offspring ---
        print(f"[gen {gen}] evaluating {config.POPULATION_SIZE - config.ELITISM_COUNT} offspring ...")
        for ind in new_pop[config.ELITISM_COUNT :]:
            ind.fitness = compute_fitness(ind, problems)
            print(f"  {ind.uid}  fitness={ind.fitness:.4f}  ops={ind.operators}")

        population = new_pop

    # --- final stats ---
    population.sort(key=lambda x: x.fitness, reverse=True)
    fits = [x.fitness for x in population]
    history.append({
        "generation": config.GENERATIONS,
        "best": max(fits),
        "mean": sum(fits) / len(fits),
        "worst": min(fits),
        "best_uid": population[0].uid,
        "best_prompt": population[0].prompt,
    })
    print(
        f"[gen {config.GENERATIONS}] FINAL  best={max(fits):.4f}  "
        f"mean={sum(fits)/len(fits):.4f}"
    )

    return population[0], history
\end{lstlisting}

\begin{warnbox}{compute\_fitness is expensive}
One call to \fn{compute\_fitness} makes \textbf{100 API calls} to Claude Haiku.
Over a full run (15 seeds + 20 gens x 13 offspring = 275 evaluations):
$275 \times 100 = 27{,}500$ Haiku API calls.  This is why the fitness pool is fixed
at 100 problems (not 1{,}000) and why Haiku (not Sonnet) is the target model.
\end{warnbox}

\begin{lstlisting}[style=example]
# population (sorted by fitness, example values):
# [ind_A: 0.82, ind_B: 0.78, ind_C: 0.74, ind_D: 0.71, ...]

# rng.sample(population, 3) draws 3 WITHOUT replacement:
contestants = [ind_C: 0.74, ind_B: 0.78, ind_D: 0.71]

# max(..., key=lambda ind: ind.fitness) returns the best:
winner = ind_B  # fitness 0.78

# Called TWICE per offspring:
p1 = tournament_select(population, rng)  # e.g., ind_B (0.78)
p2 = tournament_select(population, rng)  # e.g., ind_A (0.82)
# p1 and p2 can be the same individual if drawn twice; this is rare
# and produces a clone (crossover of identical parents = copy).
\end{lstlisting}

\begin{conceptbox}{Tournament selection: exploration vs.\ exploitation}
If we always selected the globally best individual as a parent, all offspring would
be children of the top 1-2 prompts --- the population would converge too quickly
(premature convergence) and lose diversity.  Tournament selection gives every
individual a chance proportional to its fitness: the best usually wins, but weaker
individuals can win if they happen to be drawn when even weaker contestants are in
their tournament.
\end{conceptbox}

\begin{lstlisting}[style=chain]
run(seed_prompts, problems, on_generation):

  [SETUP]
  rng = random.Random(42)  # seeded for reproducibility
  history = []

  [GEN 0: Evaluate seeds]
  population = [Individual(prompt=p) for p in seed_prompts]  # 15 individuals
  for ind in population:                                       # 15 iterations
      ind.fitness = compute_fitness(ind, problems)            # 100 API calls each

  [GENS 1..20: The evolutionary loop]
  for gen in range(1, 21):                                     # 20 iterations

      [RECORD STATS FOR PREVIOUS GENERATION]
      population.sort(by fitness, descending)
      stats = {generation: gen-1, best: ..., mean: ..., worst: ..., ...}
      history.append(stats)
      if on_generation: on_generation(gen-1, population, stats)
      # => saves generation_(gen-1).json checkpoint

      [ELITISM: copy top 2 unchanged]
      new_pop = population[:2]   # best 2 individuals copied verbatim

      [BREED: fill remaining 13 slots]
      while len(new_pop) < 15:
          p1 = tournament_select(population, rng)  # pick 2 parents
          p2 = tournament_select(population, rng)
          child = breed(p1, p2, rng)               # crossover + mutations
          new_pop.append(child)                     # child.fitness = -1.0

      [EVALUATE: only the 13 new offspring]
      for ind in new_pop[2:]:                       # skip the 2 elites
          ind.fitness = compute_fitness(ind, problems)  # 100 API calls each

      population = new_pop   # elites (re-used) + 13 new evaluated offspring

  [FINAL STATS]
  population.sort(by fitness, descending)
  history.append({generation: 20, best: ..., mean: ..., ...})

  return (population[0], history)   # best individual + all 21 stat records
\end{lstlisting}

\begin{conceptbox}{History record structure}
The \fn{history} list contains 21 entries (generations 0 through 20).  Each entry is:
\begin{lstlisting}[style=example]
{
    "generation": 0,          # or 1, 2, ..., 20
    "best": 0.7312,           # highest fitness in population at this gen
    "mean": 0.6841,           # average fitness
    "worst": 0.5920,          # lowest fitness
    "best_uid": "a3f7c2b1",   # uid of the current best individual
    "best_prompt": "Think like a competitive programmer..."
}
\end{lstlisting}
Generations 0--19 are recorded at the \emph{start} of the following generation's loop
(i.e.\ \texttt{stats["generation"]} = \texttt{gen - 1}).  Generation 20's stats are
recorded after the loop ends.
\end{conceptbox}

\begin{conceptbox}{Generational replacement}
Each generation, the \emph{entire} population is replaced (except the 2 elites).
This is called \textbf{full generational replacement}.  The alternative is
\emph{steady-state}, where only 1--2 individuals are replaced per step.
Full replacement creates strong selection pressure: any individual not in the top 2
must produce offspring that beat their parents to survive.  This speeds up convergence
at the cost of some diversity.  Elitism (top 2 guaranteed) prevents this from becoming
too aggressive.
\end{conceptbox}

def evaluate_on_holdout(best_prompt: str, holdout: list[dict]) -> float:
    """Evaluate the best evolved prompt on the held-out test set."""
    total = 0.0
    for i, prob in enumerate(holdout):
        response = llm.generate_code(best_prompt, prob["question"], prob["fn_name"])
        code = evaluator.extract_code(response)
        score = evaluator.run_tests(
            code, prob["fn_name"], prob["inputs"], prob["outputs"], config.EXEC_TIMEOUT,
        )
        total += score
        if (i + 1) % 50 == 0:
            print(f"  holdout progress: {i+1}/{len(holdout)}")
    return total / len(holdout)

#   i=0, word="yoda" => word[0] = 'y'  -- wait, 'y' not 'o'?

#   i=0, word="yoda" => word[0] = 'y'  ... but expected 'o'
#   Wait -- "enumerating order of the word" means n=index of the word, not char.
#   i=0 => word[0] from "yoda" = 'y'

#   This means: word 0 -> take letter at index 1 (0+1)?

#   Or: 'yoda'[1]='o', 'best'[2]='e'... wait 'best'[2]='s' not 'e'
#   Or: 'yoda'[0+1?]...
#   Actually checking: 'oes': o from yoda (index 1?), e from best (index 1?), s from has (index 1?)

\chapter*{Glossary}
\addcontentsline{toc}{chapter}{Glossary}

\begin{description}[style=nextline, leftmargin=3cm]

\item[Chromosome]
  A candidate solution in a genetic algorithm.  In this project, each chromosome is
  a natural-language system prompt (a paragraph of English instructions).  Stored as
  an \fn{Individual} object in \pyfile{ga.py}.

\item[Crossover]
  A genetic operator that combines two parent chromosomes into one offspring.  Here,
  the Optimizer LLM (Sonnet 4.6) \emph{semantically blends} the two parents rather
  than doing a mechanical string cut.  Fires with probability 0.8 per offspring.

\item[Deterministic (temperature = 0)]
  When a Claude model is called with \texttt{temperature=0.0}, the same input always
  produces the same output.  This is why the Target model uses temperature 0: fitness
  scores are repeatable and comparable across generations.

\item[Elitism]
  Preserving the top $k$ individuals unchanged into the next generation, preventing
  loss of the best prompt discovered so far.  \fn{ELITISM\_COUNT = 2} in this project.

\item[Evolution pool]
  The fixed set of 100 Codewars introductory coding problems used to evaluate
  fitness during the GA.  Chosen by deterministic shuffle (seed 42) of all qualified
  APPS problems.

\item[Exponential backoff]
  A retry strategy where the wait time doubles with each failed attempt.
  Used in \fn{llm.\_call()} for rate-limit errors: 5s, 10s, 20s.

\item[Fitness]
  A numeric score in [0.0, 1.0] measuring how good a chromosome is.
  Fitness = mean partial-credit score across 100 evolution problems.
  Computed by \fn{ga.compute\_fitness()}.

\item[\texttt{fn\_name}]
  The Python function name that the LLM must write.  Stored in each APPS problem's
  \texttt{input\_output.json}.  Used to construct the user message and to verify
  the generated code (\fn{evaluator.has\_function()}).

\item[Generation]
  One cycle of the GA: record stats $\to$ carry elites $\to$ select parents
  $\to$ breed offspring $\to$ evaluate offspring $\to$ replace population.

\item[Generational replacement]
  Replacing the entire population each generation (except elites), as opposed
  to steady-state (replacing 1--2 individuals at a time).  Creates strong
  selection pressure.

\item[Held-out set]
  200 problems reserved for evaluating the final evolved prompt.  Never seen
  during evolution.  Analogous to a test set in machine learning.

\item[Individual]
  One member of the GA population.  A Python \texttt{@dataclass} with fields:
  \fn{prompt}, \fn{fitness}, \fn{uid}, \fn{parent\_a}, \fn{parent\_b}, \fn{operators}.

\item[Lazy initialisation]
  Creating an object only when it is first needed.  Used for the Anthropic client
  in \pyfile{llm.py}: \fn{\_client} is \texttt{None} at module load; created on
  the first \fn{\_call()}.

\item[LLM]
  Large Language Model --- a neural network trained on text that can generate code,
  merge prompts, etc.  This project uses Claude Haiku 4.5 (target) and
  Claude Sonnet 4.6 (optimizer).

\item[Meta-prompt]
  A prompt sent to the Optimizer LLM instructing it to modify another prompt.
  The crossover and mutation functions in \pyfile{llm.py} use meta-prompts to
  direct the Optimizer.

\item[MTok]
  Million tokens --- the billing unit for LLM APIs.  The pricing formula is:
  cost = (input\_tokens $\times$ rate\_in + output\_tokens $\times$ rate\_out) / 1{,}000{,}000.

\item[Mutation]
  A genetic operator that modifies a single chromosome.  Three independent types:
  inject (add strategy sentence, 25\%), delete (remove weakest sentence, 15\%),
  rephrase (reword, 10\%).

\item[Partial credit]
  Scoring a problem as (tests passed / total tests) rather than binary all-or-nothing.
  Provides a smooth gradient for the GA to optimise.  Computed by \fn{evaluator.run\_tests()}.

\item[pass@1]
  The standard code-generation evaluation metric: fraction of problems where the LLM
  generates correct code on its first (and only) attempt.

\item[Population]
  The set of all 15 chromosomes alive at a given generation.  Maintained as
  \texttt{list[Individual]} in \fn{ga.run()}.

\item[Seed prompt]
  One of the 15 hand-crafted prompts in \pyfile{seeds.py} that form the generation-0
  population.  Each emphasises a different coding strategy.

\item[Sentinel]
  A unique string prefix (\texttt{\_\_R\_\_:}) used to tag test results in the subprocess's
  stdout, so they can be found among any stray \texttt{print()} output from generated code.

\item[Singleton]
  A class of which only one instance exists.  The \fn{tracker} object in
  \pyfile{cost\_tracker.py} is a module-level singleton; all modules that import it
  share the same instance.

\item[System prompt]
  An instruction paragraph sent \emph{before} the user message in a Claude conversation,
  shaping the model's behaviour across all subsequent messages.  In this project,
  the system prompt is the GA chromosome.

\item[Temperature]
  An LLM parameter controlling output randomness.  0.0 = fully deterministic;
  higher values introduce more variation.  Target model uses 0.0; Optimizer uses 0.7.

\item[Tournament selection]
  A parent-selection strategy: sample $k$ individuals randomly (without replacement)
  and return the fittest among them.  \fn{TOURNAMENT\_SIZE = 3} in this project.
